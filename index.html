Git, c'est quoi exactement ?

Un système de contrôle de versions distribué qui garde la trace de chaque évolution de votre code tout en favorisant la collaboration.

Historique complet

Chaque commit capture un instantané de votre projet. Revenez en arrière, explorez, comparez et expérimentez sans crainte.

Travail distribué

Tout le monde dispose d'une copie du dépôt. Même hors ligne, vous continuez à travailler et vous fusionnez quand vous êtes prêt.

Concepts clés à maîtriser

Appuyez-vous sur ces notions pour comprendre la logique de Git. Dépliez chaque carte pour obtenir une illustration synthétique.

Commit

Un commit enregistre un état du projet. Il doit être petit, clair et accompagné d'un message qui explique pourquoi vous avez fait ce changement.

Index (staging)

Zone tampon entre vos fichiers et l'historique. Vous choisissez précisément ce qui partira dans le prochain commit.

Branche

Une branche pointe vers une suite de commits. Travaillez dessus pour développer une fonctionnalité, puis fusionnez-la.

Merge vs rebase

Le merge garde l'historique complet en ajoutant un commit de fusion. Le rebase rejoue vos commits sur une base à jour pour un historique linéaire.

Un workflow moderne en cinq temps

Adoptez un rythme clair pour collaborer sans friction. Chaque étape renforce la qualité et la traçabilité de vos évolutions.

  1. 1

    Planifier

    Identifiez la fonctionnalité à livrer, créez une branche dédiée et clarifiez l'objectif avant de coder.

  2. 2

    Développer

    Progressez par petits commits cohérents. Vérifiez leur lisibilité et gardez un message clair.

  3. 3

    Documenter

    Décrivez votre travail dans la pull request, listez les impacts et capturez les points à surveiller.

  4. 4

    Tester

    Exécutez vos tests ou vérifications manuelles. Corrigez, puis ajoutez un nouveau commit si nécessaire.

  5. 5

    Partager & fusionner

    Poussez la branche vers le dépôt distant, ouvrez une demande de fusion et sollicitez un avis. Après validation, fusionnez dans la branche principale, supprimez la branche de travail et diffusez la mise à jour.

Vos routines essentielles

  • Synchronisez votre branche avant de démarrer pour éviter les mauvaises surprises.
  • Commencez vos messages de commit par un verbe à l'infinitif, clair et concis.
  • Documentez les décisions importantes dans la description de la pull request.
  • Supprimez les branches fusionnées pour garder le dépôt lisible.

Questions fréquentes

Besoin de clarifier un comportement Git ? Dépliez ces réponses rapides.

Quand utiliser git stash ?

Lorsque vous devez changer de branche sans commiter, stashez vos modifications en cours. Vous pourrez les réappliquer plus tard.

Rebase ou merge ?

Utilisez un merge pour tracer l'origine des branches et un rebase pour garder un historique linéaire. Sur du travail partagé, préférez le merge.

Comment éviter les conflits repentants ?

Récupérez souvent les nouveautés de la branche principale, communiquez avec votre équipe et découpez vos tâches en petites unités.
